Quatre structures de base

  • Liste […] — ordonnée, modifiable, doublons permis
  • Tuple (…) — ordonné, immuable
  • Ensemble {…} — non ordonné, sans doublons
  • Dictionnaire {clé: valeur} — paires étiquetées

Listes

bandes = ["bleu", "vert", "rouge", "PIR"]
print(bandes[0], "|", bandes[-1], "|", bandes[1:3])   # index, tranche
bandes.append("SWIR")
print([b.upper() for b in bandes])                    # compréhension de liste

Fonctions

def ndvi(nir, rouge):
    """NDVI = (PIR - Rouge) / (PIR + Rouge)."""
    return (nir - rouge) / (nir + rouge)

print(round(ndvi(0.6, 0.2), 3))
0.5

Le NDVI reviendra au chapitre 3 — ici c’est juste l’occasion d’introduire def/return.

Programmation objet

class Image:
    def __init__(self, capteur, bandes):   # constructeur + attributs
        self.capteur = capteur
        self.bandes = bandes
    def resume(self):                       # méthode
        return f"{self.capteur} — {self.bandes} bandes"

print(Image("Landsat-8", 11).resume())
Landsat-8 — 11 bandes

Un avant-goût de NumPy

Une image = un tableau de nombres (ndarray), la brique de tous les chapitres.

import numpy as np
image = np.array([[10, 12, 11,  9],
                  [ 8, 20, 22,  7],
                  [ 9, 21, 23,  8]])
print("Forme :", image.shape, "| max :", image.max(), "| moyenne :", image.mean().round(2))
print(image[:2, :2])          # découpage d'une sous-image
Forme : (3, 4) | max : 23 | moyenne : 13.33
[[10 12]
 [ 8 20]]

Points clés

  • Installez Python via un environnement virtuel (conda / venv)
  • Quatre structures : listes, tuples, ensembles, dictionnaires
  • if/for/while + fonctions (def) structurent le code
  • Une image est avant tout un tableau NumPy
